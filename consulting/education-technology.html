Publishing¶

https://en.wikipedia.org/wiki/Publishing

  • Project Code and Artifact Repositories
    • Version Control: Git, Mercurial
    • Project Forge: GitHub, BitBucket, SourceForge
    • Artifacts: built packages, reports, PDFs, data files
  • DOI-granting repositories (“getting a citable identifier which resolves to a URL” (~like a shorturl)):
    • https://en.wikipedia.org/wiki/Digital_object_identifier (URN)
    • https://guides.github.com/activities/citable-code/
    • https://figshare.com
    • https://zenodo.org/
  • Data Hosting (see: Web Distribution):
    • HTTP Object Storage (private/public CDN)
    • BitTorrent
      • BitTorrent + HTTP Web Seeds
    • It is faster to run the code next to the data.
      • HDFS (Hadoop, Hive, Cassandra, Spark, Mesos), DDFS (Disco)
      • SQL (Relational Database)
      • SPARQL (Linked Data)
  • Publishing Tools
    • Sphinx
    • IPython Notebook / Jupyter Notebook: nbformat JSON -> HTML, HTML slides, LaTeX, PDF, ePub, MOBI

Jupyter and Learning¶

Jupyter Project is great for learning and education.

  • Jupyter Notebook (IPython Notebook)

  • SciPy Stack (see: Pip, Conda, Anaconda)

  • Jupyter Notebook supports over 42 languages other than Python.

  • Jupyter notebooks can be published as HTML, PDF, ePub, MOBI.

  • Jupyter notebooks can be published as reveal.js HTML slide presentations.

  • Jupyter notebooks can be published to and served directly from GitHub repos.

  • Jupyter notebooks can be published as edX courses.

    • http://mail.scipy.org/pipermail/ipython-dev/2015-February/015911.html
    • https://github.com/topocm/topocm_content/
    • edX is also mostly written in Python

  • Jupyter notebooks can be saved to and read from Google Drive:

    https://github.com/jupyter/jupyter-drive

  • Jupyter notebooks are great for taking notes:

    https://github.com/notablemind/notablemind

  • Per-student, per-class, per-project Docker containers (and folders)

  • Reproducibility, Repeatability (see: Data Science)

    “Ten Simple Rules for Reproducible Computational Research”
    http://journals.plos.org/ploscompbiol/article?id=10.1371/journal.pcbi.1003285#s4

    • [ ] “Rule 3: Archive the Exact Versions of All External Programs Used”

      • [ ] Which Packages are necessary?

        • watermark: datetime stamp, package versions

          https://github.com/rasbt/watermark

        • version_information: OS, python package versions

          https://github.com/jrjohansson/version_information

      • [ ] Which Packages are instaleld?

        • Pip: pip freeze
        • Conda: conda env export
        • Dpkg: dpkg-query -l, dpkg --get-selections, wajig list-installed

      • [ ] Which OS?

        • OS X, Linux: uname -a
        • Windows: systeminfo

      • [ ] Which :ref: processor architecture? (i386, i686, x86-64, ARMv)

        • osquery
        • salt grains

      • [ ] Complete machine image (Virtualization, Provis)

        • Machine image process:

          • [ ] Take snapshot
          • [ ] Post-process (compress, add metadata, test decompression)
          • [ ] share/store/backup

        • Machine Imaging Tools:

          • Clonezilla (backup and restore partitions

            from CD/DVD, LAN, HTTP, SSH, PXE)

          • Bup (Git-like backups for

            very many / very large files)

          • Rsync, Rsnapshot, rdiff

        • Virtualization Machine Imaging Tools

          • Docker Dockerfile and image
          • Packer config and image
          • Vagrant Vagrantfile and box

  • Principle of least privilege (“privilege separation”)

    https://en.wikipedia.org/wiki/Principle_of_least_privilege

    • https://github.com/ipython/ipython/wiki/Install:-Docker
    • https://github.com/jupyter/jupyterhub/wiki/Spawners (JupyterHub)

ipython_nose¶

Source: git https://github.com/taavi/ipython_nose

ipython_nose is an extension for IPython Notebook (and TODO probably Jupyter Notebook) for discovering and running test functions starting with test_ (and TODO unittest.TestCase test classes with names containing Test) with Nose.

Note

x = 2

# these get compiled out with python -O2
assert x == 2

# these always run
# self = unittest.TestCase
self.assertEqual(self, x, 2)

if x != 2:
    ### Exceptions
    err = ('...', x)
    # raise Exception(err)
    # raise ValueError(err)


    ### Logging
    import logging
    log = logging                       # global logger
    log = __import__('logging')         # global logger
    log = logging.getLogger(__name__)   # local logger (<__name__>.py)
    log.debug(err)
    log.info(err)
    log.error(err)
    log.exception(err)
    log.critical(err)
    log.fatal(err)
    # see: supervisor.loggers.LevelsByName and .LevelsByDescription
    TRAC = LevelsByName.TRAC = 5
    BLAT = LevelsByName.BLAT = 3
    log.log(err, TRAC)
    logging.addLevelName(TRAC, 'TRAC')
    logging.addLevelName(BLAT, 'BLAT')
    log.setLevel(TRAC)
    log.log(err, BLAT)  # not printed because msg.level < log.level

    # see also: logbook, structlog, awesome-python-testing

nbgrader¶

Source: git https://github.com/jupyter/nbgrader

Jupyter notebooks can be submitted and centrally graded with nbgrader.

Note

While it’s possible to run tests of all code cells
in a Jupyter notebook, by
it’s usually preferable to factor

testable code into a module and a package (e.g. Python Package, Conda package) and then reference those tested functions from within Jupyter Notebook.

runipy¶

Source: git https://github.com/paulgb/runipy
PyPI: https://pypi.python.org/pypi/runipy

runipy runs Jupyter notebooks from a Shell commandline, generates HTML reports, and can write errors to stderr.

Jupyter notebook manual test review process:

!jupyter notebook
# - (optional) click 'Cell' > 'All Output' > 'Clear'
# - click 'Cell' > 'Run All'
!git diff mynotebook.ipynb
!git add mynotebook.ipynb
!git commit mynotebook.ipynb -m "TST: mynotebook: tests for #123"

Jupyter notebook runipy review process:

!runipy mynotebook.ipynb
web ./mynotebook.ipynb.html
for (i, o) in notebook:
    human.manually_review((i, o))
!git diff
!git diff mynotebook.ipynb*
!git add mynotebook.ipynb*
!git commit mynotebook.ipynb* -m "TST: mynotebook: tests for #123"
  • An example of runipy usage in a Makefile: https://github.com/westurner/notebooks/blob/gh-pages/Makefile
